# Virtue 첫인상 포지셔닝 스냅샷

## 0. 목적

Virtue 정식 출시 전, 신규 방문자가 첫 화면에서 **"이건 무엇이고 / 누구를 위한 것이고 / 첫 세션에서 어떤 약속을 하는지"** 가 얼마나 빨리 읽히는지 한 장으로 정리한다. 외부 발송이나 트래킹 변경 없이, 내부 문서로만 다음을 정의한다.

- 3개의 포지셔닝 가설 (누구에게 어떤 제품으로 읽히는지)
- 첫인상 리스크 (NN/g 관점에서 새 방문자가 머뭇거릴 지점)
- 코어 경험을 겪은 소수에게 나중에 물을 PMF 질문 4개 초안
- 첫 화면 헤드라인/보조문구 후보 3개 (각각 어떤 오디언스 가설에 대응하는지 명시)

본 문서는 카피 변경 지시서가 아니다. 변경 적용은 별도 Intent에서 합의 후 진행한다.

## 1. 현재 첫인상 입력값 정리

대시보드(`/`)에서 새 방문자가 처음 마주치는 시각 요소를 위에서 아래 순으로 정리한다. 이 목록이 첫인상 해석의 기반이다.

| 순서 | 요소 | 현재 문구/구성 | 신규 방문자 관점 해석 |
|---|---|---|---|
| 1 | 오늘 날짜 + Greeting | 예: `2026년 5월 19일 화요일` + `오늘 1덕만 쌓아볼까요?` (시간대별 변주) | "1덕"이라는 단어가 첫 줄에 등장 — 내부 용어. 무엇을 쌓는지 아직 미정. |
| 2 | 나의 덕력 카드 | `나의 덕력` + 큰 숫자(`AnimatedNumber`) + `이번 달 +N덕 · 어제 +N덕` | 게임처럼 누적되는 점수가 있는 듯. 다만 "덕" 단위가 무엇인지 첫 화면에 설명 없음. |
| 3 | 환생종 카드 | 이모지 + `지금은 · Lv.N` + 종 이름 + `nextHint`(`다음 생: 베일 속`) + 진행률 바 | 게임 메타포(레벨/진화/도감)는 인식되지만, "환생"이 종교적인지 게임적인지 첫 화면만으로는 모호. |
| 4 | CTA | `오늘 덕 쌓기` (카메라 아이콘 + 보라 버튼) | 행동을 강하게 유도. 다만 "사진을 찍는 행위 = 무엇" 인지 첫 클릭 전엔 추측만 가능. |
| 5 | 최근 덕행 카드 | `최근 덕행` 헤더 + 빈 상태(`아직 기록이 없어요. 오늘 사소한 거 하나, 카메라로 콕.`) 또는 직전 3건 미리보기 | 빈 상태 카피가 "사소한 거 하나"라는 가장 명확한 단서를 제공. 그러나 위 4개 요소를 먼저 읽고 도달. |

새 방문자가 첫 1초 안에 만나는 단어는 압축하면: **"덕 / 덕력 / 환생종 / 다음 생 / 오늘 1덕"**.
이 단어 묶음은 인사이더에게는 한 컷에 세계관이 전달되지만, 외부 방문자에게는 "덕(德)"의 추상도가 높아 제품 범주를 판단하기 전에 한 번 더 추리를 요구한다.

## 2. 포지셔닝 가설 3개

April Dunford 식으로 각 가설은 **대안(컨텍스트) → 차별점 → 가치 → 대상 고객**의 조합으로 정리한다. 가설은 서로 배타적이지 않지만, "첫 화면이 가장 먼저 누구를 알아보게 만드는가" 기준으로 우선순위가 다르다.

### 가설 A — "사소한 일상의 결을 사진으로 보관하는 자기관찰 게임"

- 컨텍스트(대안 카테고리): 감사일기 앱, 무드 트래커, 한 줄 일기. 사용자는 이미 "오늘 하루 어땠는지 잠깐 들여다보고 싶은" 욕구를 갖고 있다.
- 차별점: 글이 아니라 **사진 + 짧은 메모 + 가벼운 AI 코멘트**. 무거운 회고나 도덕적 칭찬 없이 사실 묘사 + 농담 1줄.
- 가치 약속: "오늘 좀 괜찮은 인간이었나"를 30초 안에 가볍게 확인. 누적된 결이 다음 생의 환생종으로 시각화되어 작은 게임적 보상이 따라온다.
- 대상 고객: 감사일기·무드 트래커를 시도해봤지만 "글 쓰기 싫음 / 진지함이 부담스러움"으로 이탈한 20–30대. 자기 검열을 약하게 하고 싶은 사람.
- 첫 화면 적합도: 현재 첫 화면에서 가장 강하게 읽힌다. "최근 덕행" 빈 상태 카피(`오늘 사소한 거 하나, 카메라로 콕.`)가 이미 이 톤에 가장 가깝다.

### 가설 B — "내가 만든 ‘오늘 1덕’ 미니 게임 (라이프 RPG)"

- 컨텍스트(대안 카테고리): Habitica, Finch, 스트릭 기반 미니 게임, 출석 체크형 앱. 사용자는 "행동의 누적이 캐릭터/세계관을 키운다"는 메타포에 친숙하다.
- 차별점: 캐릭터 육성이 아니라 **"다음 생의 환생종"** 이라는 단발 메타포. 종 진화는 11단계로 끝이 있고, 도감(`/dex`)이 컬렉션 욕구를 흡수한다. 일일 30덕 상한·재채점 3회 한도로 과몰입을 사전에 차단.
- 가치 약속: 무거운 운영 비용 없이도 매일 1덕씩 누적되는 "내 인생의 작은 잔고". 환생종이 바뀌는 순간은 모달로 잠깐 축하.
- 대상 고객: 게임적 요소를 좋아하지만 "헬창/생산성 종교"가 부담스러운 사람. Brush Up Life 드라마처럼 가벼운 환생 메타포에 끌리는 사람.
- 첫 화면 적합도: 환생종 카드 + 진행률 바가 강하게 이 가설을 지지한다. 다만 "덕"의 의미가 정의되기 전에 게임화 신호가 먼저 도착하면 제품 진심이 가벼워 보일 위험.

### 가설 C — "AI가 매일 보는 셀프 거울 — 진지하지 않은 자기 인식 루틴"

- 컨텍스트(대안 카테고리): Replika·Pi 같은 AI 동반자, 캐주얼 코칭 챗봇, 일일 셀프체크 앱. 사용자는 "AI가 내 일상에 한 마디 해주는" 경험을 이미 인지하고 있다.
- 차별점: 채팅이 아니라 **사진 한 장 + 짧은 채점**. AI는 점수와 한 줄 코멘트만 돌려준다. 칭찬/설교 금지, 사실 묘사 + 농담 1줄(`copy-spec.md` 금지어 명단으로 보호).
- 가치 약속: 30초 동안 "오늘 내가 했던 작은 행동을 외부 시선으로 한 번 본다". 결과지가 아니라 거울에 가깝다.
- 대상 고객: AI 챗봇을 한 번 써봤지만 "대화가 길어서 부담스러움"으로 이탈한 사람. 자기 인식에 관심 있지만 코칭 앱은 무거운 사람.
- 첫 화면 적합도: 현재 첫 화면에는 AI 신호가 약하다(`/add` 페이지에서야 등장). 이 가설을 살리려면 보조문구 한 줄에 AI를 명시할 필요가 있을 수 있다(§5 후보 3 참조).

### 3개 가설 비교 표

| 항목 | A 자기관찰 게임 | B 라이프 RPG | C AI 셀프 거울 |
|---|---|---|---|
| 대안 카테고리 | 감사일기, 무드 트래커 | Habitica, 출석체크 | AI 동반자, 셀프체크 챗봇 |
| 핵심 차별점 | 사진 + 농담 코멘트 | 환생종 진화 + 30덕 상한 | 사진 한 장 → AI 한 줄 |
| 첫 세션 약속 | 30초 안의 가벼운 회고 | 1덕 = 다음 생의 잔고 | AI가 보는 오늘의 한 컷 |
| 첫 화면 적합도 | 높음 (현재 톤과 가장 가까움) | 중 (환생종 카드 강함, "덕" 모호) | 낮음 (AI 신호는 `/add`에서야 등장) |
| 위험 | 너무 일상적이라 게임감 약화 | 게임감이 진심을 가볍게 만들 수 있음 | AI를 너무 앞세우면 사진 행동이 부담됨 |

권장 우선순위: **A를 메인, B를 보조, C는 보조문구에서만 살짝 신호**. 그 이유는 §5 헤드라인 후보에서 다시 정리한다.

## 3. 첫인상 리스크 (NN/g 관점)

NN/g는 첫인상이 **관련성·신뢰·사용성** 판단에 자동적으로 영향을 준다고 본다. 새 방문자가 첫 화면에서 머뭇거릴 지점을 4개로 정리한다.

### R1 — "덕"이 무엇인지 즉시 해석되지 않음

- 증상: 첫 줄 인사 `오늘 1덕만 쌓아볼까요?`, 두 번째 카드 `나의 덕력`. "덕"의 한자(德) 추상도가 높아 신규 방문자는 "선행? 미덕? 게임 점수?"를 추리해야 한다.
- NN/g 매핑: 관련성 판단 지연 → 신뢰 형성 감소.
- 카피-스펙 충돌 점검: "미덕"·"선행"은 금지어 명단에 포함(`copy-spec.md` §금지어 명단 — "무거움. 덕/덕행으로 충분"). 즉, **금지어로 쉽게 풀어 설명할 수 없다.** 대신 보조문구 한 줄에서 "오늘 사소한 행동"이라는 비-도덕 단어로 풀어줘야 한다.

### R2 — "환생"이 종교 신호인지 게임 신호인지 첫 화면만으로 모호

- 증상: 환생종 카드, `다음 생: 베일 속`, 도감 메타포. 외부 방문자는 "이건 종교 콘텐츠인가, 게임인가"를 판단하기 전에 한 번 더 추리한다.
- NN/g 매핑: 관련성 + 신뢰 동시 흔들림. 종교 신호로 읽히면 일부 사용자가 이탈, 게임 신호로만 읽히면 진심이 가벼워 보임.
- 완화 방향: README가 이미 명시한 **"Brush Up Life 드라마에서 영감 / 라이프 게임"** 톤을 첫 화면 보조문구에도 한 번 노출하면 모호함이 줄어든다. (단, 외부 IP 언급은 헤드라인 전면이 아니라 README/스플래시 등 보조 영역에 한정.)

### R3 — CTA가 "사진을 찍는 행위 = 무엇"인지 첫 클릭 전엔 추측만 가능

- 증상: `오늘 덕 쌓기` 버튼은 강하지만, 첫 클릭 전에는 "사진 = 인증 / 자랑 / 일기 / 점수 매기기" 중 무엇인지 알 수 없다.
- NN/g 매핑: 사용성 + 신뢰. 너무 이른 권한 요청(카메라/갤러리)이 신뢰를 깎을 수 있다는 NN/g 지적과 직결.
- 완화 방향: 버튼 자체보다 **버튼 위/아래의 한 줄 보조문구**가 "사진 = 오늘 사소한 행동 한 컷"이라는 의도를 노출하면 충분. 카피-스펙은 이미 `/add` 페이지의 사진 placeholder에 같은 톤을 권장(`오늘의 한 컷, 골라주세요`).

### R4 — 가입/알림/권한 요청이 첫 화면에 강하게 등장하지는 않지만, 첫 세션 안의 카메라 권한이 시작점이 됨

- 증상: 현재는 첫 화면에 가입·이메일·푸시 권한 요청이 없다(localStorage 기반). 단, `오늘 덕 쌓기`를 누르면 즉시 카메라/갤러리로 이어진다.
- NN/g 매핑: 너무 이른 권한 요청은 신뢰를 깎는다. 다행히 현재 구조는 "탐색 → CTA → 권한"으로 단계가 분리되어 있어 비교적 안전.
- 완화 방향: 변경 불필요. **단, 첫 세션 안에 가입/이메일/푸시를 추가하는 변경은 별도 Intent로 분리하고 본 스냅샷의 가정 밖**으로 둔다.

### 리스크 요약

R1·R2가 메시지 리스크(말로 풀어야 함), R3·R4가 행동 리스크(설계로 풀어야 함). 본 스냅샷은 R1·R2를 우선 처리 대상으로 본다. R3는 보조문구로 일부 흡수 가능, R4는 변경 없음.

## 4. PMF 질문 4개 초안

Sean Ellis/Superhuman 방식을 따르되, **본 문서는 질문지를 만드는 단계까지만** 한다. 외부 발송·설문·인터뷰 모집은 별도 승인이 필요한 Out of scope다.

대상: 코어 경험을 겪은 소수 = **앱을 설치하고 1주일 안에 `deed_saved`가 최소 1회 발생한 사용자**(`seven-day-deed-loop.md` §1 정의에 부합).

질문 의도: 40%라는 숫자 자체보다, "매우 아쉽다"고 답한 사람과 그들이 말한 benefit/사용 맥락을 분리해 들기.

### PMF-1 (메인) — Sean Ellis 본 질문

- 문장: `Virtue가 내일부터 사라진다면 얼마나 아쉬울까요?`
- 옵션: `① 매우 아쉽다 / ② 조금 아쉽다 / ③ 아쉽지 않다 / ④ 이미 안 쓰고 있다`
- 분석 의도: ①을 답한 소수만 분리해 PMF-2~PMF-4의 자유 응답을 본다. ②·③·④는 평균에 섞지 않는다.
- 톤 점검: 명령조 없음, 격언 없음, 도덕 칭찬 없음. **카피-스펙 §금지어 명단과 충돌 없음.**

### PMF-2 — "누구에게 가장 잘 맞을 것 같나요"

- 문장: `Virtue가 가장 잘 맞을 친구나 지인 한 명을 떠올린다면, 그 사람은 어떤 사람인가요?`
- 보조: `한 줄로 묘사해주세요. 직업/관심사/평소 습관 중 떠오르는 것 아무거나.`
- 분석 의도: April Dunford 식 "대상 고객"을 사용자 언어로 듣는다. 가설 A/B/C 중 어느 쪽이 ①답변자들에게 가장 강하게 읽혔는지 본다.
- 톤 점검: "좋은 사람" 같은 인격 평가 단어 없음. 친구·지인이라는 일상 단어로 환원.

### PMF-3 — "어떤 점이 가장 좋았나요"

- 문장: `Virtue를 쓰면서 가장 마음에 든 점은 무엇이었나요?`
- 보조: `짧게 적어주세요. 한 줄도 좋아요.`
- 분석 의도: 코어 가치(benefit)를 사용자 자신의 단어로 듣는다. 헤드라인 후보 §5의 단어 선택을 사후 검증할 1차 재료.
- 톤 점검: "훌륭한"·"본받을" 같은 도덕 칭찬 없음. "마음에 든 점"은 부드러운 만족도 질문.

### PMF-4 — "그게 없으면 어떻게 대체하실 건가요"

- 문장: `Virtue가 사라진다면, 비슷한 걸 어떤 다른 방법으로 대신하실 것 같아요?`
- 보조: `없다면 ‘없다’도 괜찮아요.`
- 분석 의도: 사용자가 인식하는 **진짜 대안 카테고리**를 듣는다. 가설 A(감사일기/무드 트래커)·B(Habitica/출석체크)·C(AI 동반자) 중 어느 쪽이 실제 대안으로 떠오르는지 본다. April Dunford의 "대안" 정의 단계.
- 톤 점검: 명령조 없음. "없다도 괜찮아요"가 응답 부담을 낮춤.

### 질문 운용 규칙 (Out of scope이지만 미리 못박음)

- 본 4개 질문은 **외부 발송 전 별도 승인**이 필요하다. 메일·SMS·푸시·웹훅·DM 어느 채널이든 마찬가지.
- 보낸다면 ①답변자만 분리해 PMF-2~4를 보는 게 원칙. ①이 임계치 미만이면 숫자 해석을 보류한다.
- "응답 안 함"이 정상이다. 응답률 자체를 PMF 신호로 해석하지 않는다.

## 5. 헤드라인 / 보조문구 후보 3개

각 후보는 §2의 가설 중 하나에 1:1 대응한다. **위치는 첫 화면 Greeting 줄 또는 그 바로 아래 보조문구 한 줄**을 가정한다. CTA(`오늘 덕 쌓기`)는 유지.

> 공통 톤 가드: 카피-스펙(`copy-spec.md`)의 금지어 명단(훌륭한/본받을/모범적인/귀감/인성/미덕/선행/베풂/봉사정신/마음이 따뜻한/좋은 사람/멋진 인격), 격언·명언 인용 금지, 사용자 직접 칭찬 금지, 종교 어휘는 메타포 한정.

### 후보 1 — 가설 A(자기관찰 게임)에 대응

- 헤드라인: `오늘, 사소한 거 하나 찍어둘까요?`
- 보조문구: `30초면 충분해요. 작은 행동 한 컷이 누적되는 라이프 게임.`
- 의도: 첫 줄에서 "사진 = 사소한 행동 한 컷"이라는 코어 의도를 즉시 노출. 보조문구가 "라이프 게임"으로 카테고리 신호를 한 번 더 준다.
- 금지어 충돌 점검: "사소한", "한 컷", "라이프 게임" — 모두 카피-스펙 금지어 명단에 없음. "오늘 1덕만 쌓아볼까요?"의 기본 폴백과 톤 호환됨.

### 후보 2 — 가설 B(라이프 RPG)에 대응

- 헤드라인: `오늘 1덕, 다음 생에 적립.`
- 보조문구: `매일 한 칸씩 차오르는 환생종. 무리 안 해도 11단계까지.`
- 의도: 환생종 카드와 직접 연결되는 게임 메타포를 헤드라인 전면에 둔다. 보조문구가 "무리 안 해도"라는 단어로 30덕 상한·재채점 한도 같은 유연성 신호를 살짝 흘린다.
- 금지어 충돌 점검: "1덕", "다음 생", "환생종", "한 칸씩 차오르는" — 모두 카피-스펙·README 어휘 내. 종교 어휘("다음 생", "환생종")는 게임 메타포 한정이라 카피-스펙 §추가 규칙 준수. "적립"은 기존 `judge.ts`에 이미 사용된 어휘.
- 주의: "다음 생"이 헤드라인 전면에 오면 R2(환생/종교 모호)가 강화될 수 있어, 후보 1보다 우선순위 낮음.

### 후보 3 — 가설 C(AI 셀프 거울)에 대응

- 헤드라인: `오늘의 한 컷, AI한테 한 번 보여줄까요?`
- 보조문구: `점수는 1~5덕. 사실 묘사 + 농담 한 줄. 칭찬도 설교도 없음.`
- 의도: AI 시그널을 첫 화면에 명시. 보조문구가 카피-스펙의 톤 약속("사실 묘사 + 가벼운 농담", 도덕 칭찬/설교 금지)을 사용자에게 직접 약속한다.
- 금지어 충돌 점검: "한 컷", "AI", "점수", "사실 묘사", "농담", "칭찬도 설교도 없음" — 모두 카피-스펙 금지어 명단에 없음. **"칭찬도 설교도 없음"이라는 표현은 칭찬/설교를 행위로 부정하므로 금지어 사용이 아니다.**
- 주의: AI를 첫 화면에서 강조하면 R3(사진 행동의 부담)가 약하게 강화될 수 있다. 단, 카메라 권한이 CTA 클릭 이후라 R4 리스크는 낮은 편.

### 후보 3개 비교 표

| 후보 | 대응 가설 | 카테고리 신호 | R1 완화 | R2 완화 | 카피-스펙 충돌 |
|---|---|---|---|---|---|
| 1 `오늘, 사소한 거 하나 찍어둘까요?` | A | "라이프 게임" | 강 (사소한 행동) | 중 (환생 미언급) | 없음 |
| 2 `오늘 1덕, 다음 생에 적립.` | B | "환생종 적립" | 중 (1덕 직접 사용) | 약 (다음 생 강조 시 모호 ↑) | 없음 (메타포 한정) |
| 3 `오늘의 한 컷, AI한테 한 번 보여줄까요?` | C | "AI 셀프 체크" | 강 (한 컷) | 강 (환생 미언급) | 없음 |

권장: 첫 화면 한 줄에 한 후보만 노출. **메인은 후보 1, A/B 분기 테스트가 가능해지면 후보 3과 비교**, 후보 2는 환생종 카드 보조 라벨에서 일부만 활용.

## 6. First verification gate (충돌 점검)

### 6.1 카피-스펙 금지선과의 충돌 점검 (필수 게이트)

| 카피-스펙 금지선 | 본 문서 충돌 여부 | 근거 |
|---|---|---|
| 도덕적 칭찬 (훌륭한/멋진 인격/좋은 사람/마음이 따뜻한 등) | **없음** | §2·§4·§5 어느 후보에도 인격 평가 단어 없음. PMF-3은 "마음에 든 점" 만족도 질문이라 인격 칭찬 아님. |
| 설교 (본받을/모범적인/귀감/봉사정신 등) | **없음** | 후보 카피는 모두 행동 묘사 + 효과 짚기. 사용자에게 "이래야 한다"고 말하지 않음. |
| 격언/명언/속담 인용 금지 | **없음** | 헤드라인·보조문구 어느 후보에도 인용 없음. |
| 무거운 단어 (미덕/선행/베풂/인성) | **없음** | "사소한 행동", "한 컷", "1덕" 등 카피-스펙 어휘 풀 내에서만 단어 선택. |
| 사용자 직접 칭찬 (당신은 ~한 사람) 금지 | **없음** | 모든 후보가 "오늘 ~ 할까요?" 형식의 권유 또는 행동 묘사로 작성. |
| 종교 어휘는 메타포 한정 | **준수** | "다음 생", "환생종"은 README에 명시된 게임 메타포 한정. 후보 2 주의 메모에서 우선순위를 낮춰둠. |

결론: **본 문서의 모든 카피 후보는 `docs/copy-spec.md` 금지선과 충돌하지 않는다.** 적용 단계에서는 §금지어 명단을 1회 더 재확인하는 것을 권장한다.

### 6.2 README/seven-day-deed-loop와의 충돌 점검

| 항목 | 충돌 여부 | 근거 |
|---|---|---|
| 5탭 구조(`/`, `/add`, `/deeds`, `/dex`, `/me`) | 없음 | 본 문서는 첫 화면 한 줄/보조문구만 다룸. 라우트·탭 변경 제안 없음. |
| 로컬 저장(localStorage) 가정 | 없음 | PMF 질문은 외부 발송용이며 앱 내 구현 변경을 요구하지 않음. |
| 30덕 상한 / 재채점 3회 한도 | 없음 | 후보 2 보조문구가 "무리 안 해도"로 같은 방향을 강화. 한도 자체 변경 없음. |
| `deed_saved` 중심 첫 7일 루프 | 없음 | PMF 대상 정의를 `seven-day-deed-loop.md` §1에 맞춤. 새 이벤트 정의 없음. |
| 채점 모드(mock/ai) 기본값 | 없음 | 후보 3은 모드와 무관하게 적용 가능 (AI 폴백 시 mock으로 자연 흐름). |

### 6.3 외부 발송/배포/트래킹/생산/시크릿/권한/공개 액션 비수행 명시 (필수)

본 문서를 작성하고 커밋하는 과정에서 **다음 액션은 단 한 건도 수행하지 않았다.**

- 외부 발송: 이메일·SMS·푸시·웹훅·DM·SNS·설문 어떤 채널로도 사용자/지인/팀 외부에 메시지를 보내지 않았다.
- 배포: 어떤 운영 환경(`https://virtue.oracle.shdkej.com` 포함)에도 새 빌드를 올리지 않았다. CI 트리거 없음.
- 트래킹: PostHog 또는 그 외 어떤 분석 도구에도 새 이벤트·속성·플래그를 추가/수정하지 않았다.
- 생산 코드: `src/`·`api/`·`public/`·런타임 설정 어느 것도 본 Intent로 변경되지 않았다. 본 변경은 `docs/`·infinity 문서 한정.
- 시크릿/권한: `.env*`·시크릿 매니저·API 키·OAuth/IAM/리포 권한 어느 것도 본 Intent로 추가/회전/변경되지 않았다.
- 공개 액션: PR·이슈·공개 코멘트·외부 리포 변경·외부 URL 게시 어느 것도 본 Intent로 수행되지 않았다. git push는 사내 내부 리포지토리(`virtue-rebirth-app`·`prompt-archive`) 한정.

## 7. Out of scope (별도 승인 필요)

- 헤드라인/보조문구 후보의 첫 화면 실제 반영 (코드 변경, 톤 토글 분기, A/B 테스트).
- PMF 질문 4개의 외부 발송, 사용자 모집, 응답 수집, 응답 분석 인프라.
- PostHog 새 이벤트/속성/대시보드 작성.
- 카메라 권한 흐름 변경, 가입/이메일/푸시 도입.
- 외부 IP(드라마/책/논문) 인용을 헤드라인 전면에 노출.

위 항목은 모두 별도 Intent에서 다룬다.
