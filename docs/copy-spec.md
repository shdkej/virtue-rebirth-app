# 한국어 카피 델타 (product-01)

> 톤: 따뜻 + 장난기. 설교/도덕적 칭찬 금지. 기본 부드러운 존댓말.
> "오늘 좀 괜찮은 인간이었나" 자기 만족 톤. 사실 묘사 + 가벼운 농담 1줄.
> 이 문서는 코드 변경 없이 카피 델타만 제안합니다. 적용은 별도 에이전트가 수행.

## 페이지별 카피 델타

### 대시보드 (`/`)

#### 상단 인사 (시간대별 변주)

현재: `오늘 1덕만 쌓아볼까요?`

| 시간대 | 시각 가이드 | 제안 문구 |
|---|---|---|
| 아침 | 05:00–10:59 | `오늘 1덕만 쌓아볼까요?` (유지) |
| 낮 | 11:00–16:59 | `점심은 드셨어요? 1덕도 같이.` |
| 저녁 | 17:00–21:59 | `퇴근하셨어요? 가벼운 1덕 어떠세요?` |
| 심야 | 22:00–04:59 | `오늘 하루, 한 덕 정도는 있었을 거예요.` |
| 폴백 (시간 모름) | — | `오늘 1덕만 쌓아볼까요?` |

> 구현 메모: `new Date().getHours()` 기반 분기. 시간 모르면 폴백.

#### "나의 덕력" 라벨

- 유지: `나의 덕력` — 이 앱의 정체성 단어. 바꾸지 않는 게 좋음.
- 추가 제안 (sub-라벨 한 줄, 카드 좌측 하단 등):
  - `다음 생을 위한 적금 같은 것` — 단, 화면이 빡빡하면 생략.

#### `이번 달 +83 · 어제 +4` 친근 변주

| 현재 | 제안 |
|---|---|
| `이번 달 +83 · 어제 +4` | `이번 달 +83 · 어제 +4` (유지: 사실 묘사가 톤에 맞음) |
| — | 변주 A: `이번 달 +83덕 · 어제 +4덕` (단위 명시) |
| — | 변주 B: `이번 달 +83 · 어제 +4 · 그저께 ?` (옵션, 가벼운 농담 — 데이터 있을 때만) |

권장: 변주 A. 단위가 살짝 더 따뜻함.

#### 환생종 카드 보조 라벨

현재: `환생종 · Lv.3`

| 현재 | 제안 | 비고 |
|---|---|---|
| `환생종 · Lv.3` | `지금은 · Lv.3` | 종 이름이 바로 옆에 있으니 "환생종"은 중복 |
| — | `머무는 단계 · Lv.3` | 도감 페이지와 어휘 통일 |

권장: `지금은 · Lv.3` (간결 + 따뜻).

#### `다음: ??? 잠금` 티저

현재: `다음: ??? 잠금`

| 제안 | 톤 |
|---|---|
| `다음 단계: 비공개` | 담백 |
| `다음 생: 베일 속` | 살짝 장난 |
| `다음: ??? · 곧` | 미니멀, 호기심 |

권장: `다음 생: 베일 속` — 도감과 연결되고 톤 일치.

#### CTA `오늘 덕 쌓기`

- 유지 권장. 짧고 명료하며 톤 OK.
- 옵션 (시간대별 — 화려해질 수 있어 비권장):
  - 심야 한정: `자기 전에 1덕`

#### 빈 상태 (no deeds yet)

신규 제안 (최근 덕행 섹션 빈 상태):
- 메인: `아직 기록이 없어요.`
- 서브: `오늘 사소한 거 하나, 카메라로 콕.`

## 톤 토글 가이드

### soft (부드러운 존댓말, 기본값)

- 어미: `~요` / `~네요` / `~군요` / `~죠` / `~어요`
- 설교 금지. 자기 만족 + 사실 묘사 + 가벼운 농담 1줄.

예시 5개:
1. `사진 속 장면은 작지만 또렷한 친절이네요.`
2. `이웃집 강아지도 안심했을 거예요.`
3. `월요일 아침의 카페인은 거의 인도주의죠.`
4. `오늘은 충분히 쌓았어요. 내일 또 봐요.`
5. `전화 한 통이면 충분한데 그게 제일 어렵죠.`

### casual (장난기 있는 반말)

- 어미: `~네` / `~다` / `~지` / `~군` / `~어`
- 친구가 살짝 놀리듯. 명령조/반복 욕설 금지.

예시 5개:
1. `사진 속 장면은 작지만 또렷한 친절이네.`
2. `이웃집 강아지도 안심했겠다.`
3. `월요일 아침의 카페인은 거의 인도주의지.`
4. `오늘은 그만. 내일 또 보자.`
5. `전화 한 통이 제일 어렵다는 거, 알지.`

## 금지어 명단 (재확인)

다음 어휘는 어떤 카피에서도 등장하지 않도록 합니다.

| 금지어 | 사유 |
|---|---|
| 훌륭한 | 도덕적 칭찬 |
| 멋진 인격 | 도덕적 칭찬 |
| 좋은 사람 | 인격 평가 |
| 본받을 | 설교 |
| 마음이 따뜻한 | 인격 평가 |
| 모범적인 | 설교 |
| 귀감 | 설교 |
| 인성 | 인격 평가 |
| 미덕 | 무거움. "덕"으로 충분 |
| 선행 | 무거움. "덕행"으로 충분 |
| 베풂 | 시혜적 |
| 봉사정신 | 학교적 |

추가 규칙:
- 격언/명언/속담 인용 금지. (예: "콩 심은 데 콩 난다" 같은 류 금지.)
- 종교적 어휘 사용 시 메타포 한정. 설교 금지. (예: "다음 생" OK, "업보 쌓아라" NG.)
- 사용자를 직접 칭찬하지 않음. 행동을 묘사하거나 효과를 짚음. (예: "잘 했어요"는 가끔 OK, "당신은 좋은 사람이에요"는 NG.)

## Operator 관점 마이크로카피

운영자(현재는 1인 개발자 본인) 입장에서 나올 만한 시스템 카피.

| 상황 | 제안 |
|---|---|
| 사진 업로드 실패 | `사진을 못 불러왔어요. 한 번 더 시도해주세요.` |

## 적용 우선순위 (구현 에이전트용)

1. **HIGH (톤 일관성)**: 대시보드 환생종 카드 보조 라벨 (`환생종` → `지금은`), 잠긴 단계 표기 (`잠금` → `???` + blurb), species stage 4/5 blurb 미세 조정.
2. **MID (UX 따뜻함)**: `/add` 사진 placeholder, 메모 placeholder, 안내 한 줄.
3. **LOW (장식)**: 시간대별 인사 변주, 토스트 메시지 풀.
4. **신규 추가**: 빈 상태 메시지 (대시보드 / 덕행록), 30덕 상한 도달 메시지, 에러 카피.

> 핵심 원칙: 톤 일관성 > 변화량. 너무 많이 손대면 본래의 담백함이 사라짐.
