# Virtue 첫 7일 덕행 루프 정의서

## 0. 목적

정식 출시 전, 첫 7일 동안 사용자가 "다시 열 이유"를 어디서 얻는지 한 장으로 정의한다. 구현은 별도 승인 후. 본 문서는 행동 정의 + 카피 후보 + 유연성 원칙 + 측정 후보까지만 다룬다.

## 1. 셀 행동 정의 (루프 단위)

핵심 셀 행동은 `deed_saved`. 하루에 1번 이상 발생하면 그날 루프가 돈 것으로 본다. 보조 신호는 의도와 마찰을 본다.

| 이벤트 | 역할 | 해석 |
|---|---|---|
| `deed_saved` | 핵심 | 하루 1건 이상이면 루프 성공. 7일 중 며칠 발생했는지로 첫 주 리텐션을 본다. |
| `deed_judged` | 보조 (의도) | 채점은 했는데 저장이 없다면 "결과가 기대와 달랐다" 신호. judged - saved 갭을 본다. |
| `deed_rerolled` | 보조 (마찰) | 한 사이클 안 3회 한도. 평균 재채점 횟수가 높으면 첫 결과의 톤/점수가 기대와 어긋남. |
| `deed_save_capped` | 보조 (과몰입) | 첫 주에 30덕 상한이 자주 닿으면 오히려 피로 신호. 일반적이지 않을 가능성. |

루프 한 바퀴: `덕 입력 → 채점 확인 → 저장 → 다음 날 다시 열 이유 확인`.

## 2. 첫 1·3·7일 마일스톤 카피 후보

모두 앱 내 노출. 외부 발송 없음. 톤은 카피-스펙 준수 (soft 기본, casual 변주). 숫자 streak 표기는 첫 주에는 사용하지 않는다.

### Day 1 — 첫 저장 직후

위치: `/add` 저장 완료 토스트 직후, 또는 `/` 대시보드 진입 시 상단 한 줄.

- soft 후보 A: `오늘 1덕, 잘 챙겨뒀어요. 내일 또 한 컷 부탁해요.`
- soft 후보 B: `첫 덕 들어왔어요. 두 번째가 제일 가벼워요.`
- casual 변주: `첫 덕, 챙겼다. 두 번째가 제일 가볍다.`

원리: Calm 사례의 "핵심 가치 경험 직후" 타이밍 차용. 단 알림이 아니라 같은 화면 안 한 줄.

### Day 3 — 2→3일 성장감 구간

위치: `/deeds` 진입 시 상단 부제, 또는 `/` 대시보드 "이번 주 N번 기록" 라벨.

- soft 후보 A: `이번 주 벌써 세 번째. 흐름이 생겼네요.`
- soft 후보 B: `2일에서 3일은 의외로 큰 한 칸이에요.`
- casual 변주: `세 번째 덕. 흐름 생긴 거 맞다.`

원리: Duolingo가 강조한 초반 2→3 성장감. 숫자 streak 없이 "세 번째"로만 표현.

### Day 7 — 첫 주 마무리

위치: `/deeds` 상단 주간 요약, 또는 `/` 환생종 카드 하단 보조 라벨.

- soft 후보 A: `한 주 동안 덕이 차곡차곡. 다음 주는 더 가볍게 가요.`
- soft 후보 B: `7일째 들여다본 사람, 생각보다 적어요. 한 주 잘 했어요.`
- casual 변주: `한 주 마감. 다음 주는 더 가볍게.`

원리: Duolingo의 마일스톤 피드백 개선이 7일 리텐션 +1.7%. 작은 감정 보상의 위치를 7일째에 명시.

## 3. 유연성·실패 처리 원칙

빠진 날을 죄책감 없이 다룬다. 도덕 압박 금지.

### 원칙 A — 빠진 날은 셈하지 않는다

- 근거: Duolingo Streak Freeze가 일일 활성 +0.38%. 빠질 여지를 주는 게 활성을 높였다.
- Virtue 적용: 첫 7일에는 "X일 연속" 같은 가시 카운터를 노출하지 않는다. 대신 "이번 주 N번"으로 셈한다.
- 카피·동작 힌트: `어제 비었어도 오늘 한 덕은 그대로예요.` (저장 직후 한 줄, 직전 날 빈 상태일 때만 노출.)

### 원칙 B — 실패는 마찰 신호로만 본다

- 근거: BJ Fogg B=MAP. 행동 미발생은 의지 부족이 아니라 동기·쉬움·프롬프트 중 하나의 부재.
- Virtue 적용: 빠진 다음 날 첫 진입에서 사용자를 다그치지 않고 가장 쉬운 1덕부터 권한다.
- 카피·동작 힌트: `오랜만이에요. 사진 하나면 충분해요.` (대시보드 상단 인사 폴백으로만, 빈도 제어 필요.)

## 4. B=MAP 매핑

- Motivation: "오늘도 조금 괜찮은 인간이었나"를 가볍게 확인하고 싶은 자기 만족.
- Ability: 사진 1장 + 메모 한 줄, 30초. 30덕 상한은 과몰입 방지.
- Prompt: 외부 알림 없음. 앱 진입 시 상단 한 줄(시간대별 인사 + 마일스톤 카피)이 유일한 프롬프트.

## 5. PostHog 측정 후보

이미 구현된 이벤트는 그대로 활용. 신규는 후보만 표시. 새 이벤트 구현 지시는 본 문서 범위 밖.

| 이벤트 | 상태 | 용도 |
|---|---|---|
| `deed_saved` | 기존 | D1/D3/D7 리텐션 본 지표. |
| `deed_judged` | 기존 | judged - saved 갭으로 첫 결과 만족도 추정. |
| `deed_rerolled` | 기존 | 평균 재채점 횟수로 첫 채점 톤 적합도 추정. |
| `deed_save_capped` | 기존 | 30덕 상한 도달 빈도. 과몰입/피로 신호. |
| `level_up_viewed` | 기존 | 첫 주 안 환생종 변화 도달률 (부가 동기). |
| `first_session_deed_saved` | candidate | 첫 세션 안 첫 저장까지 도달률. 기존 `deed_saved`에 `is_first` 속성 추가로 대체 가능. 신규 이벤트보다 속성 추가가 가볍다. |
| `weekly_active_days` | candidate (계산형) | 인사이트 단계에서 `deed_saved`를 user/day로 distinct count. 새 이벤트 불필요. |
| `milestone_copy_viewed` | candidate | D1/D3/D7 카피가 실제 노출됐는지 확인. 노출 카피와 행동의 인과를 보고 싶을 때만. 우선순위 낮음. |

## 6. First verification gate (충돌 점검)

- README와 충돌: 없음. 5탭 구조, 로컬 저장, 30덕 상한, 재채점 3회 한도 모두 본문 가정과 일치.
- copy-spec와 충돌: 없음. 카피 후보는 금지어 회피, soft/casual 톤 분리, 격언 미사용 준수.
- posthog-setup-report와 충돌: 없음. 기존 이벤트만으로 D1/D3/D7 리텐션과 마찰 신호를 다 잡을 수 있다. 신규는 candidate로만 표시.

## 7. Out of scope (본 문서 범위 밖, 별도 승인 필요)

- 외부 푸시·이메일·SMS·웹훅 발송.
- 친구 공유, 외부 SNS 노출, 공개 프로필.
- 숫자 streak UI 구현, streak 깨짐 보상(Freeze 등) 구현.
- 새 PostHog 이벤트 구현, 트래킹 인프라 변경.
- 30덕 상한 변경, 재채점 횟수 변경, 요금/유료화.

위 항목은 본 문서로 결정하지 않는다.
