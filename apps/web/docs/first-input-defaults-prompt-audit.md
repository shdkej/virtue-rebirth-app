# Virtue 첫 입력 기본값 / 예시 / placeholder 감사표

## 0. 목적과 한 줄 요약

AI/agent 제품에서 **기본 프롬프트·placeholder·예시(suggested question)는 사용자의 첫 요청을 강하게 형성한다**(출처노트 §핵심요약). 제품이 무엇을 묻도록 권하느냐가 "사용자가 어떤 모드로 제품을 이해하는가"를 사실상 설계한다. 기능 설명형 기본값은 사용자를 support-bot 모드로 끌고 가고, 사용자의 실제 맥락에 붙은 기본값은 후속 행동을 더 잘 만든다.

Virtue의 `/add`는 대화형 agent가 아니지만, **첫 덕행 입력 유도(사진 슬롯·메모 placeholder)와 AI 판정 결과**가 같은 역할을 한다. 본 문서는 현재 `/add`(및 그 앞단 `/` 홈)에 실제로 존재하는 첫 입력 유도 문구를 잡별(J1~J4)로 한 표에 모아, **"기능 구경형 / 기능 설명형 / 잡 수행형" 중 어디에 치우쳤는가**를 내부 감사한다.

> **핵심 발견 한 줄:** Virtue의 첫 입력 기본값에는 *예시 덕행·추천 질문·카테고리 칩이 전혀 없다*. 유일한 입력 유도는 (a) 사진 슬롯 placeholder와 (b) 사용자의 실제 행동을 묻는 중립적 질문형 메모 placeholder(`뭐 했어요?`)뿐이다. 이는 support-bot 모드로 끌려갈 위험은 없다는 점에서 출처노트의 "기능 설명형 회피" 권고와 우연히 정렬하지만, **동시에 잡별 조향(steering)이 0이라 같은 중립 placeholder가 J1~J4를 구분 없이 부른다.** 이 중립성의 양면을 잡별로 읽는 것이 본 감사의 본론이다.

전제 못박기:

- Virtue는 정식 출시 전이고 decision-grade 실사용 지표가 없다. 모든 잡과 후속 행동 해석은 **가설**이며 손기록 관찰 렌즈다.
- 본 문서는 **새 트래킹·이벤트·속성·코드·카피를 만들지 않는다.** 후속 행동 지표는 이미 구현된 PostHog 이벤트만 인용한다.
- first value 매핑은 선행 문서를 그대로 계승한다: **J1/J2/J4 = `deed_saved`, J3 = `deed_judged`**(저장 전). 재정의 0.

## 1. 첫 입력 유도 인벤토리 (현재 코드에 실재하는 것만)

`/add`와 그 앞단 `/` 홈에 실제로 존재하는 첫 입력 유도 요소를 file:line으로 고정한다. **존재하지 않는 것(예시·추천 질문)도 명시**해야 감사가 정직하다.

| # | 표면 | 유형 | 현재 문구 (실재) | 위치 | Amplitude 렌즈 분류 |
|---|------|------|------------------|------|---------------------|
| D1 | `/` 홈 빈 카드 | 유도 CTA 문장 | `아직 비어있어요. 오늘 1덕만 시작해볼까요?` | `src/app/page.tsx:67` | 잡 수행형(작게 시작 권유) |
| D2 | `/` 홈 최근 덕행 빈 상태 | 유도 제목+설명 | `아직 기록이 없어요.` / `오늘 사소한 거 하나, 카메라로 콕.` | `src/app/page.tsx:128-129` | 잡 수행형(행동 예시 1개 암시) |

| D9 | `/add` 하단 힌트 | 안내 문장 | `사진은 이 기기에만 잠시 머물러요. 키가 설정돼 있으면 AI가, 없으면 mock이 채점해요.` | `src/app/add/page.tsx:407-408` | 신뢰/안심형(기능 설명 일부) |

**존재하지 않는 것(감사의 핵심 공백):**

- 추천 질문 / suggested prompt 칩 — **없음.**
- 예시 덕행("예: 분리수거 했어요", "예: 엄마한테 안부 전화") — **없음.** D2의 `카메라로 콕`이 행동 1개를 암시할 뿐 입력란에 예시를 채우지 않는다.
- 카테고리/태그 선택지(입력 전) — **없음.** 태그는 판정 *결과*로만 등장(`add/page.tsx:352-360`).
- "무엇을 할 수 있나 설명해줘" 류 기능 설명형 기본 프롬프트 — **없음.** Virtue는 대화 agent가 아니라 입력 슬롯이라 support-bot 유도 문구 자체가 구조적으로 부재.
- 미리 채워진(pre-filled) 입력값 — **없음.** 사진·메모 모두 빈 상태로 시작.

> 즉 Virtue의 첫 입력 기본값은 **"질문형 placeholder + 빈 슬롯"** 단일 패턴이다. 출처노트가 경계한 "기능 설명형 기본값"은 구조적으로 없지만, "사용자의 실제 데이터·목표에 붙은 product-focused prompt"도 D6 한 줄(`뭐 했어요?`)에 그친다.

## 2. J1~J4 × 첫 입력 기본값 감사표 (문서의 심장)

각 행은 한 잡. 컬럼은 `이 잡을 부르는 현재 기본값 / 그 기본값의 조향 강도 / 좋은 후속 행동(이벤트) / 위험(잘못된 모드로 끌림) / 관찰 손기록 질문`. 모든 셀은 가설이며 §1 인벤토리에 근거한다. 잡 정의는 `first-session-jtbd-matrix.md` §1을 계승한다.

| 잡 | ① 이 잡을 부르는 현재 기본값(D-코드) | ② 조향 강도(이 기본값이 그 잡으로 모는 힘) | ③ 좋은 후속 행동 = first value 이벤트 | ④ 위험: 잘못된 모드로 끌림 | ⑤ 관찰 손기록 질문 |
|---|---|---|---|---|---|
| **J1 기록형**<br>"사소한 행동을 남기고 싶다" | D2(`사소한 거 하나, 카메라로 콕`)·D3(`오늘 덕 쌓기`)·D4 사진 슬롯·D6(`뭐 했어요?`) | **중상.** "남길 대상"을 직접 지목(사진+한 줄). D6 질문이 행동 회상을 바로 유도. | 첫 입력 → 판정 → **`deed_saved`**(남겼다=저장으로 닫힘). 판정은 통과점. | D7 `AI 채점`이 전면에 보이면 "남기러 왔는데 채점을 기다리는" 마찰(J1엔 대기=나쁜 마찰, 매트릭스 §4 계승). | 첫 입력이 D6 질문에서 바로 나왔나, 아니면 사진부터 골랐나? 저장까지 멈춤 없이 갔나? |
| **J2 누적형**<br>"얼마나 쌓였는지 게임처럼 보고 싶다" | D1(`오늘 1덕만 시작`)·D3(`덕 쌓기`)·홈 누적 카드(첫 진입 `count===0`이면 비어 있음) | **하.** 첫 입력 기본값 자체는 누적을 약속하지 않음. "나중에 보려고"라는 말이 나왔나? |

### 표의 핵심 판독

1. **D6(`뭐 했어요? 한 줄이면 충분해요.`)이 유일한 잡-중립 product-focused 프롬프트다.** 출처노트가 권한 "사용자의 실제 행동/맥락을 묻는 프롬프트"에 정확히 해당하고, "기능 설명형"이 아니다 — 이건 좋은 기본값이다. 다만 *중립적*이라 J1~J4를 구분해 부르지 않는다.
2. **첫 입력 단계에서 가장 약하게 불리는 잡은 J3와 J2다.** J3는 AI 약속이 `/add` *안*에서야(그것도 ai 모드에서만) 등장하고, J2는 누적 보상이 두 번째 저장 이후라 첫 입력 기본값이 약속하지 못한다.
3. **환경 분기(D7/D8)는 J3 첫 인상을 흔든다.** mock 모드의 `임시 판정` 라벨은 "AI가 어떻게 볼지"라는 J3 동기를 첫 입력 단계에서 *낮춘다*. 이는 카피 문제가 아니라 런타임 모드 문제다(코드 사실만 기록, 변경 제안 아님).

## 3. 후속 행동(두 번째 행동) 관찰 후보

출처노트의 핵심 신호는 "첫 메시지보다 **첫 응답 뒤 두 번째 메시지로 이어지는가**"였다(§핵심요약). Virtue에는 대화의 "두 번째 메시지"가 없으므로, 이를 **잡별 두 번째 행동**으로 번역한다.

| 잡 | "두 번째 행동" 번역 | 인용 이벤트(기존) | 정상/이탈 판독 |
|---|---|---|---|
| **J1** | 판정 → **저장** | `deed_judged` → `deed_saved` | 저장 없이 닫힘 = 저장 전 이탈 후보 |
| **J2** | 저장 → (누적 확인) 두 번째 저장/환생종 도달 | `deed_saved` → `level_up_viewed` | 첫 세션 1회 저장은 정상, 누적은 첫 주 관찰 |
| **J3** | 판정 → **다시 판정 / 다른 입력**(저장 강요 안 함) | `deed_judged` → `deed_rerolled` 또는 다른 입력 후 `deed_judged` | **저장 없이 `deed_rerolled`/다른 입력 = 정상(J3 잡 충족)**. judged−saved 갭을 이탈로 단정 금지 |
| **J4** | 판정 → **저장**(메모 중심 항목) | `deed_judged` → `deed_saved` | 저장 없이 닫힘 = 저장 전 이탈 후보 |

**첫 10~20명 손기록 표에 추가할 칸 후보(구현 아님, 관찰 양식 제안):**

- `첫 입력 출처`: D6 placeholder를 보고 썼나 / 사진부터 골랐나 / 자유 입력 / 외부 설명(친구가 알려줌) 중 무엇이었나.
- `후속 행동`: `deed_rerolled` / 다른 입력 후 재판정 / `deed_saved` / 저장 없이 닫음 중 무엇이었나.

> 이 두 칸은 `first-real-user-baseline-template.md`·`first-60-second-value-observation-script.md`의 기존 관찰 행에 **재사용·증설**한다. 새 표·새 텔레메트리·새 속성을 만들지 않는다.

## 4. Prelaunch 금지선 (decision-grade 아님)

- **첫 입력 출처/후속 행동 손기록을 전환율·activation rate·retention%·PMF·벤치마크로 환산하지 않는다.** 출처노트의 Amplitude 수치는 참고 사례이지 Virtue 목표치가 아니다(출처노트 §Source Boundary).
- **한 명의 첫 입력/후속 행동으로 "기본값이 좋다/나쁘다"를 단정하지 않는다.** 반복 패턴만 방향 재료.
- **J3의 judged−saved 갭을 가치 부재·이탈로 읽지 않는다.** 저장 없이 `deed_rerolled`/다른 입력/닫힘은 J3 잡 충족 후 정상 종료일 수 있다(매트릭스 §3, `shareworthy`·`product-body-vs-bumper` 계승).
- **`deed_save_capped`(`add/page.tsx:167`)·503·판정 지연은 availability/friction**이며 첫 입력 기본값 품질이나 후속 행동 의지로 환산하지 않는다(`Availability And Friction Are Not Value` 계승).
- **synthetic/mock/self-test 입력(데모 시드·임시 판정·메이커 자가 테스트)은 사람 첫 입력 증거나 J3 first value 판단에 섞지 않는다**(`Traffic Source Before Metrics` 계승).
- **D7 `임시 판정`(mock) 세션의 J3 신호를 ai 모드 J3 호기심으로 합산하지 않는다.** 모드를 먼저 분리하고 읽는다.
- **변경 금지:** 신규 이벤트·속성·카피 반영·placeholder 변경·예시 추가·suggested prompt 도입·트래킹·대시보드·session replay·배포·외부 발송·비용·시크릿·권한·개인정보 변경 0. 본 문서는 `docs/` 1파일 추가만.

## 5. 후속 후보 (proposal-only, 본 Intent로 구현 금지)

아래는 *관찰로 검증한 뒤* 별도 Intent에서 결정한다. 지금 확정·구현하지 않는다.

- **J3 전용 흐름:** 저장을 강요하지 않는 재판정/다른 예시 흐름이 필요한지, 현재 문서가 아니라 첫 사용자의 반복 `deed_rerolled`·다른 입력 손기록으로 확인(출처노트 §후속 실험 후보).
- **잡별 예시 조향:** 만약 J3/J2가 첫 입력 단계에서 덜 불린다는 패턴이 반복되면, *전역* 예시를 넣어 한 잡을 살리고 다른 잡을 방해하지 말고(출처노트 §적용가능성), 잡별로 다른 예시/placeholder를 검토. 단 카피·예시 추가는 copy-spec 금지어 대조 + approval-needed.
- **mock/ai 라벨 분기:** D7/D8의 mock 라벨이 J3 첫 인상을 낮추는 문제는 런타임 모드 정책이며 마케팅 카피 변경 범위 밖. proposal-only로 기록만.

## 6. 계승 / 변경 / 충돌 분리

### 6-1. 계승한 기준 (변경 없이 그대로 사용)

- **first value 매핑:** J1/J2/J4 = `deed_saved`, J3 = `deed_judged`(저장 전). 재정의 0. (`First Value Mapping`, `first-session-jtbd-matrix.md` §2·§5)
- **마찰 부호 잡별 반전:** AI 채점 대기는 J3엔 가치, J1·J4엔 마찰. (매트릭스 §4)
- **J3 저장 없는 종료 = 정상:** judged−saved 갭은 J3 한정 자연 종료. (`shareworthy`, `product-body-vs-bumper`)
- **availability ≠ value:** `deed_save_capped`·503·지연은 friction. (`Availability And Friction Are Not Value`)
- **트래픽 분류 선행 / prelaunch 비결정 등급:** synthetic/mock·소수 표본은 방향 재료. (`Traffic Source Before Metrics`, `Prelaunch Decision Boundary`)

### 6-2. 이번에 새로 배운 것 (이 문서가 추가하는 것)

1. **Virtue의 첫 입력 기본값은 "질문형 placeholder + 빈 슬롯" 단일 패턴이고, 예시·추천 질문·카테고리 칩이 전혀 없다.** "기능 설명형 기본값"(support-bot 유도)은 구조적으로 부재 — 이건 출처노트 권고와 우연히 정렬한다.
2. **그 중립성은 양날이다:** support-bot 모드로 끌릴 위험은 0이지만, **잡별 조향도 0**이라 같은 D6(`뭐 했어요?`)가 J1~J4를 구분 없이 부른다. 첫 입력 단계에서 가장 약하게 불리는 잡은 **J3(AI 약속이 `/add` 안에서야 등장)와 J2(누적 보상이 두 번째 저장 이후)**다.
3. **환경 분기(`IS_AI_MODE`)가 J3 첫 인상을 흔든다:** mock 모드의 `임시 판정` 라벨은 "AI가 어떻게 볼지"라는 J3 동기를 첫 입력 단계에서 *낮춘다*(코드 사실, 변경 제안 아님).

### 6-3. 다음 Marketer에게 넘길 규칙

- 첫 입력을 읽을 때 **"어떤 기본값(D-코드)이 이 잡을 불렀는가"를 먼저 분리**하고, 그 다음 후속 행동(§3 표)을 본다. 첫 입력 출처를 모르면 후속 행동 해석도 흔들린다.
- **"두 번째 메시지" = 잡별 두 번째 행동**으로 번역한다: J1/J4=`deed_saved`, J2=`level_up_viewed`/두 번째 저장, **J3=`deed_rerolled`/다른 입력(저장 아님)**. J3에 저장을 두 번째 행동으로 기대하지 않는다.
- **전역 예시/placeholder 최적화 금지.** 한 잡을 살리는 예시가 다른 잡을 support-bot/구경 모드로 끌 수 있으므로, 조향이 필요하면 잡별로 검토하고 반드시 관찰로 먼저 확인한다.
